Every agent has access to 47 built-in tools. Agents can also [create custom skills](/features/agents#self-extending-skills) at runtime and use [MCP tool servers](/features/mcp).

## Execution

| Tool | Purpose |
|------|---------|
| `run_command` | Shell command execution (max timeout 300s) |

## File Operations

| Tool | Purpose |
|------|---------|
| `read_file` | Read file contents from `/data` |
| `write_file` | Write/append file in `/data` |
| `list_files` | List/glob files in `/data` |

## HTTP

| Tool | Purpose |
|------|---------|
| `http_request` | HTTP GET/POST/PUT/DELETE/PATCH |

## Browser Automation

| Tool | Purpose |
|------|---------|
| `browser_navigate` | Open URL, extract page text |
| `browser_get_elements` | Accessibility tree snapshot with element refs (e1, e2, ...) |
| `browser_screenshot` | Capture page screenshot |
| `browser_click` | Click element by ref or CSS selector |
| `browser_type` | Fill input by ref or CSS selector |
| `browser_scroll` | Scroll page up/down or scroll element into view |
| `browser_hover` | Hover over element by ref or CSS selector |
| `browser_press_key` | Press keyboard key (Enter, Escape, Tab, etc.) |
| `browser_wait_for` | Wait for element, navigation, or network idle |
| `browser_go_back` | Navigate back in browser history |
| `browser_go_forward` | Navigate forward in browser history |
| `browser_switch_tab` | Switch between open browser tabs |
| `browser_reset` | Force-close browser and reconnect fresh |
| `browser_solve_captcha` | Auto-solve CAPTCHAs on the current page |

All agents share a persistent Camoufox (stealth Firefox) browser service running in a separate container. KasmVNC serves a live browser view on port 6080.

## Memory

| Tool | Purpose |
|------|---------|
| `memory_search` | Hybrid search across workspace files and structured DB |
| `memory_save` | Save fact to workspace and structured memory DB |

## Web Search

| Tool | Purpose |

## Fleet Awareness

| Tool | Purpose |
|------|---------|

## Introspection

| Tool | Purpose |
|------|---------|
| `get_system_status` | Query own runtime state: permissions, budget, fleet, cron, health |

## Shared State (Blackboard)

| Tool | Purpose |
|------|---------|
| `read_shared_state` | Read from the shared blackboard |
| `write_shared_state` | Write to the shared blackboard |
| `list_shared_state` | Browse blackboard entries by prefix |
| `watch_blackboard` | Watch for blackboard key changes (pattern-based) |
| `publish_event` | Publish event to mesh pub/sub |
| `subscribe_event` | Subscribe to mesh pub/sub events at runtime |
| `claim_task` | Claim a task from the agent's task queue |
| `save_artifact` | Save deliverable file and register on blackboard |

All blackboard operations are subject to [permission checks](/reference/configuration#configpermissionsjson--agent-permissions).

## Scheduling

| Tool | Purpose |
|------|---------|
| `set_cron` | Schedule a recurring job (pass `heartbeat=true` for autonomous monitoring with probes) |
| `list_cron` | List scheduled jobs |
| `remove_cron` | Remove a scheduled job |

See [Triggering & Automation](/features/triggering) for details.

## Skills Management

| Tool | Purpose |
|------|---------|
| `create_skill` | Write a new Python skill at runtime |
| `reload_skills` | Hot-reload all skills |

## Agent Spawning

| Tool | Purpose |
|------|---------|
| `spawn_agent` | Spawn a new ephemeral agent in its own container via the mesh |
| `spawn_subagent` | Spawn a lightweight in-process sub-agent (no container overhead) |
| `list_subagents` | List active in-process sub-agents spawned by this agent |
| `wait_for_subagent` | Wait for a subagent to complete and return its result |

`spawn_agent` creates a fully isolated container agent via the mesh host. `spawn_subagent` creates a lightweight in-process helper — faster but shares the parent's process. Sub-agents can nest up to depth 2 (max 3 concurrent per parent).

## Credential Vault

| Tool | Purpose |
|------|---------|
| `vault_generate_secret` | Generate a secure random secret |
| `vault_capture_from_page` | Capture credentials from a browser page |
| `vault_list` | List available credentials in the vault |
