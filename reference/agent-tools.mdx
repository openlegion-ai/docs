Every agent has access to 40 built-in tools. Agents can also [create custom skills](/features/agents#self-extending-skills) at runtime and use [MCP tool servers](/features/mcp).

## Execution

| Tool | Purpose |
|------|---------|
| `exec` | Shell command execution with timeout |

## File Operations

| Tool | Purpose |
|------|---------|
| `read_file` | Read file contents from `/data` |
| `write_file` | Write/append file in `/data` |
| `list_files` | List/glob files in `/data` |

## HTTP

| Tool | Purpose |
|------|---------|
| `http_request` | HTTP GET/POST/PUT/DELETE/PATCH |

## Browser Automation

| Tool | Purpose |
|------|---------|
| `browser_navigate` | Open URL, extract page text |
| `browser_snapshot` | Accessibility tree snapshot with element refs (e1, e2, ...) |
| `browser_screenshot` | Capture page screenshot |
| `browser_click` | Click element by ref or CSS selector |
| `browser_type` | Fill input by ref or CSS selector |
| `browser_evaluate` | Run JavaScript in page |
| `browser_reset` | Force-close browser and reconnect fresh (new IP for Bright Data) |

Browser tools are pre-installed in the agent container image. Four backends are available:

| Backend | Config Value | Description |
|---------|-------------|-------------|
| **Basic** | `basic` (default) | Standard Playwright Chromium |
| **Stealth** | `stealth` | Camoufox anti-detect browser (automatic fingerprint rotation) |
| **Advanced** | `advanced` | Bright Data Scraping Browser via CDP (residential proxies, CAPTCHA solving) |
| **Persistent** | `persistent` | KasmVNC-based persistent browser with visual monitoring from the dashboard |

Configure per-agent in `config/agents.yaml`:

```yaml
agents:
  scraper:
    browser_backend: advanced
```

## Web Search

| `web_search` | Search the web via DuckDuckGo (no API key required) |

## Fleet Awareness

| Tool | Purpose |
|------|---------|
| `notify_user` | Send a notification to the user (via active channel) |
| `list_agents` | Discover other agents in the fleet |
| `read_agent_history` | Read another agent's conversation logs |

## Workspace

| Tool | Purpose |
|------|---------|
| `update_workspace` | Update identity files (HEARTBEAT.md, USER.md) |

## Introspection

| Tool | Purpose |
|------|---------|
| `introspect` | Query own runtime state: permissions, budget, fleet, cron, health |

## Shared State (Blackboard)

| Tool | Purpose |
|------|---------|
| `read_shared_state` | Read from the shared blackboard |
| `write_shared_state` | Write to the shared blackboard |
| `list_shared_state` | Browse blackboard entries by prefix |
| `publish_event` | Publish event to mesh pub/sub |
| `save_artifact` | Save deliverable file and register on blackboard |

All blackboard operations are subject to [permission checks](/reference/configuration#configpermissionsjson--agent-permissions).

## Scheduling

| Tool | Purpose |
|------|---------|
| `set_cron` | Schedule a recurring job |
| `set_heartbeat` | Enable autonomous monitoring with probes |
| `list_cron` | List scheduled jobs |
| `remove_cron` | Remove a scheduled job |

See [Triggering & Automation](/features/triggering) for details.

## Skills Management

| Tool | Purpose |
|------|---------|
| `create_skill` | Write a new Python skill at runtime |
| `list_custom_skills` | List all custom skills the agent has created |
| `reload_skills` | Hot-reload all skills |

## Agent Spawning

| Tool | Purpose |
|------|---------|
| `spawn_agent` | Spawn a new ephemeral agent in its own container via the mesh |
| `spawn_subagent` | Spawn a lightweight in-process sub-agent (no container overhead) |
| `list_subagents` | List active in-process sub-agents spawned by this agent |

`spawn_agent` creates a fully isolated container agent via the mesh host. `spawn_subagent` creates a lightweight in-process helper â€” faster but shares the parent's process. Sub-agents cannot spawn further sub-agents.

## Credential Vault

| Tool | Purpose |
|------|---------|
| `vault_generate_secret` | Generate a secure random secret |
| `vault_capture_from_page` | Capture credentials from a browser page |
| `vault_list` | List available credentials in the vault |
| `vault_status` | Check vault health and credential status |
