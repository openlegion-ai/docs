## What is OpenLegion?

OpenLegion is a platform for running autonomous AI agent fleets. Every agent gets its own isolated Docker container with private memory, tools, schedule, and budget — coordinated through deterministic YAML workflows.

API keys never leave the vault. Agents call through a proxy and never handle credentials directly. Chat with your fleet via **Telegram**, **Discord**, **Slack**, **WhatsApp**, or CLI. Monitor everything from a real-time web dashboard.

No LangChain. No Redis. No Kubernetes. No CEO agent. 100+ LLM providers.

## Why OpenLegion?

<CardGroup cols={2}>
  <Card title="Security by Architecture" icon="shield">
    Every agent runs in an isolated Docker container. API keys live in a credential vault — agents never see them. Defense-in-depth with 6 security layers.
  </Card>
  <Card title="Deterministic Orchestration" icon="diagram-project">
    YAML-defined DAG workflows with step dependencies, conditions, retries, and failure handlers. No LLM decides what runs next.
  </Card>
  <Card title="Cost Controls" icon="dollar-sign">
    Per-agent LLM token tracking with enforced daily and monthly budget caps at the vault layer. Agents physically cannot spend what you haven't authorized.
  </Card>
  <Card title="Self-Improving Agents" icon="brain">
    Agents learn from tool failures and user corrections, injecting past learnings into future sessions. They write their own Python skills at runtime and hot-reload them.
  </Card>
  <Card title="Multi-Channel" icon="comments">
    Connect agents to Telegram, Discord, Slack, and WhatsApp. Also accessible via CLI and API. Monitor via real-time web dashboard.
  </Card>
  <Card title="100+ LLM Providers" icon="plug">
    Configurable model failover chains cascade across LLM providers with per-model health tracking and exponential cooldown via LiteLLM.
  </Card>
</CardGroup>
