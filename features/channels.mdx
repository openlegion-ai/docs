Chat with your agent fleet via **Telegram**, **Discord**, **Slack**, **WhatsApp**, **Webhooks**, or **CLI**. Channels provide a unified interface for interacting with agents from any platform.

## Connecting Channels

Channels are configured through config files and environment variables (see [Channel Configuration](#channel-configuration) below). Enable a channel in `config/mesh.yaml` and set the corresponding API token environment variables.

On next `openlegion start`, a pairing code appears — send it to your bot to link. The first user to interact becomes the channel owner and can manage access for other users.

## Supported Channels

| Channel | Setup | Features |
|---------|-------|----------|
| **CLI REPL** | Built-in | `@agent` mentions, `/commands`, broadcast |
| **Telegram** | Bot token from @BotFather | Per-user agent routing, chunked messages |
| **Discord** | Bot token from Discord Developer Portal | Per-user agent routing, chunked messages |
| **Slack** | Bot token (Socket Mode — no webhooks needed) | Per-user agent routing, chunked messages |
| **WhatsApp** | Access token + phone number ID from WhatsApp Cloud API | Per-user agent routing, chunked messages |
| **Webhooks** | Webhook URL configuration in `config/mesh.yaml` | HTTP-based integration, event-driven |

## How Channels Work

All channels share a common base class that provides:
- **@agent routing**: Mention an agent by name to direct your message
- **Slash commands**: `/use`, `/agents`, `/status`, `/costs`, `/reset`, `/broadcast`
- **Per-user routing**: Each user's messages route to their selected agent
- **Message chunking**: Long responses are automatically split for platform limits

## Channel Configuration

Channels are configured in `config/mesh.yaml`:

```yaml
channels:
  telegram:
    enabled: true
    default_agent: assistant
  discord:
    enabled: false
    default_agent: assistant
  slack:
    enabled: false
  whatsapp:
    enabled: false
```

API tokens are stored as environment variables:

```bash
OPENLEGION_CRED_TELEGRAM_BOT_TOKEN=123456:ABC...
OPENLEGION_CRED_DISCORD_BOT_TOKEN=MTIz...
OPENLEGION_CRED_SLACK_BOT_TOKEN=xoxb-...
OPENLEGION_CRED_SLACK_APP_TOKEN=xapp-...
OPENLEGION_CRED_WHATSAPP_ACCESS_TOKEN=EAAx...
OPENLEGION_CRED_WHATSAPP_PHONE_NUMBER_ID=1234...
```

## Channel Commands

These commands work across all channels:

| Command | Description |
|---------|-------------|
| `@agent <message>` | Send message to a specific agent |
| `/use <agent>` | Switch your active agent |
| `/agents` | List all running agents |
| `/status` | Show agent health |
| `/costs` | Show today's LLM spend |
| `/reset` | Clear conversation with active agent |
| `/broadcast <msg>` | Send message to all agents |
| `/steer <msg>` | Inject message into a busy agent's context |
| `/debug` | Show recent request traces |
| `/addkey <svc> [key]` | Add an API credential to the vault |
| `/help` | Show available commands |

<Note>
The following commands are available only in the CLI REPL: `/add`, `/agent`, `/edit`, `/remove`, `/restart`, `/history`, `/blackboard`, `/queue`, `/workflow`, `/cron`, `/project`, `/credential`, `/removekey`, `/logs`, `/traces`
</Note>
