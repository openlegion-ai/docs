OpenLegion includes a built-in web dashboard for real-time monitoring, debugging, and management of your agent fleet. The dashboard is served by the mesh host and available at `http://localhost:8420/dashboard`.

## Panels

### Fleet (tab)

Agent cards showing health status, current model, token usage, and cost. Organized by projects. Click an agent to open a detail view with workspace editor and embedded VNC viewer for live browser sessions.

Slide-over chat panels let you chat with any agent directly from the dashboard via SSE streaming — no need to switch to the CLI or a messaging channel.

### Activity (tab)

- **Traces** — Grouped request traces showing the full lifecycle of each agent interaction, including LLM prompts, responses, tool calls, and timing. LLM prompt/response previews for quick debugging.
- **Live Events** — WebSocket-streamed event feed showing real-time activity across the fleet. Filter by event type (task assignments, pub/sub events, blackboard updates, etc.).
- **Blackboard** — Browse, search, write, and delete entries in the shared blackboard. See the current state of inter-agent shared data.
- **Costs** — Per-agent spending breakdown with period selector (today, this week, this month). Budget bars show how much of each agent's allocation has been used.
- **Automation** — Manage cron jobs and scheduled tasks. Run, pause, resume, edit, or delete automation from the UI.

Broadcast is project-scoped — messages are sent to all agents within the selected project.

### System (tab)

View configured credentials (names only — values are never exposed), pub/sub subscriptions, and model pricing.

### Keyboard Command Palette

Quick access to dashboard actions via a keyboard command palette for power users.

## Accessing the Dashboard

The dashboard starts automatically with `openlegion start`. No additional configuration is needed.

```
http://localhost:8420/dashboard
```

The dashboard uses Alpine.js and Tailwind CSS, served as static assets from the mesh host. Real-time updates are streamed via WebSocket and SSE.
