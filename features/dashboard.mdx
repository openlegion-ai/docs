OpenLegion includes a built-in web dashboard for real-time monitoring, debugging, and management of your agent fleet. The dashboard is served by the mesh host and available at `http://localhost:8420/dashboard`.

## Panels

### Agents

View all running agents with their health status, current model, token usage, and cost. Edit agent configuration and budgets directly from the dashboard. Agent identity editor lets you modify personality, instructions, preferences, heartbeat rules, memory, activity logs, and learnings.

### Activity & Traces

Grouped request traces showing the full lifecycle of each agent interaction — including LLM prompts, responses, tool calls, and timing. LLM prompt/response previews for quick debugging.

### Live Events

WebSocket-streamed event feed showing real-time activity across the fleet. Filter by event type (task assignments, pub/sub events, blackboard updates, etc.). Streaming broadcast with real-time per-agent responses.

### Slide-Over Chat Panels

Chat with any agent directly from the dashboard via slide-over panels — no need to switch to the CLI or a messaging channel.

### Blackboard

Browse, search, write, and delete entries in the shared blackboard. See the current state of inter-agent shared data.

### Costs

Per-agent spending breakdown with period selector (today, this week, this month). Budget bars show how much of each agent's allocation has been used.

### Automation

Manage cron jobs and scheduled tasks. Run, pause, resume, edit, or delete automation from the UI.

### Browser Viewer

Embedded KasmVNC viewer for agents using the `persistent` browser backend — watch agent browser sessions in real time from the dashboard.

### System

View configured credentials (names only — values are never exposed), pub/sub subscriptions, model pricing, and browser backend configuration.

### Keyboard Command Palette

Quick access to dashboard actions via a keyboard command palette for power users.

## Accessing the Dashboard

The dashboard starts automatically with `openlegion start`. No additional configuration is needed.

```
http://localhost:8420/dashboard
```

The dashboard uses Alpine.js and Tailwind CSS, served as static assets from the mesh host. Real-time updates are streamed via WebSocket.
